Tab (browse mode)

From the top of the page, pressing tab sends focus to the whole page and reads NVDA iframe test document button open menu. Interacting with the button that is read works, although focus wasn't visibly moved to it. If focus is moved back to this tab stop with Shift + Tab, nothing is read.

Pressing Tab twice from the top of the page sends focus to the iframe container and no content is read, but pressing Enter or space interacts with the first element, a button, although it does not have visible focus. Pressing Shift + Tab to move back to this tab stop does not read anything.

Pressing Tab three times sends focus to the first focusable element but does not read it. Pressing Shift + Tab back to this tab stop reads the element.

If NVDA is automatically reading content from the top of the page on load and the Tab key is pressed, focus moves as expected to the next focusable element after the element currently being read.

On the first tab, no content is read. On the second, NVDA reads the context of the first element in its body that receives focus, and the element's properties.
