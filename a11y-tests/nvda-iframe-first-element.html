NVDA iframe first element test

Stop link

Tested with Firefox 41.0 and NVDA 2015.3 on Windows 7, to see how iframes behave when accessed as the first element on the page.

The iframe output:

<iframe title="This is the iframe title attribute" src="http://simplyaccessible.com" width="420" height="300">
<p>Here is a text fallback for the iframe, with <a href="http://simplyaccessible.com">a link</a>.</p>
</iframe>

On page load (browse mode)

NVDA iframe test busy iframe banner landmark navigation landmark button open menu...

NVDA identifies the iframe, says it is busy as it accesses the iframe content, identifies the iframe again, then begins reading the iframe content from top to bottom as it would read from the top of the page normally. It does not read the title of the iframe content.

Shift + M (browse mode)

NVDA reads: This is the iframe title attribute iframe banner landmark navigation landmark open menu button when accessed from further down the page.

NVDA reads the iframe's title attribute, identifies the iframe, drills down through any ARIA role information, and reads the first element in the iframe. It does not read the title of the iframe content.

Tab (browse mode)

From the top of the page, pressing tab sends focus to the whole page and reads NVDA iframe test document button open menu. Although focus wasn't visibly moved to the button itself but to the page, interacting with the button that is read works. If focus is moved back to this tab stop with Shift + Tab, nothing is read.

Pressing Tab twice from the top of the page sends focus to the iframe container and no content is read, but pressing Enter or space interacts with the first element in the iframe, a button, although it does not have visible focus. Pressing Shift + Tab to move back to this tab stop does not read anything.

Pressing Tab three times sends focus to the first focusable element in the iframe but does not read it. Pressing Shift + Tab back to this tab stop reads the element.

If NVDA is automatically reading content from the top of the page on load and the Tab key is pressed, focus moves as expected to the next focusable element after the element currently being read.

Tab (forms mode)

NVDA reads: This is the iframe title attribute iframe this is the iframe title attribute document

NVDA reads the iframe's title attribute, identifies the iframe, speaks the iframe's title attribute a second time, then identifies the content of the iframe as a document. It does not read the title of the iframe content.

Recommendation

NVDA identifies iframes and iframe content differently depending on how users navigate to the iframe. In the case of loading an iframe as the first element on the page, iframes present particular challenges to users who are attempting to navigate via elements that recieve keyboard focus, such as skip navigation, global navigation. Since users will have different experiences based on how they access the content and users relying on the Tab key will have particularly inconsistent experiences, refraining from using iframes in general is recommended.
