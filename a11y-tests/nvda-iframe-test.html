Tested with Firefox 41.0 and NVDA 2015.3 on Windows 7, to see how iframes behave when accessed.

The iframe output:

<iframe title="This is the iframe title attribute" src="http://simplyaccessible.com" width="420" height="300">
<p>Here is a text fallback for the iframe, with <a href="http://simplyaccessible.com">a link</a>.</p>
</iframe>

Down arrow (browse mode)

When accessing the iframe via the down arrow, NVDA reads: iframe banner landmark navigation landmark button open menu

NVDA identifies the iframe, drills down through any ARIA role information, and reads the first element in the iframe. It does not read the iframe's title attribute or the title of the iframe content. Additionally, it identifies the button and then reads its label, instead of the other way around.

M (browse mode)

When accessing the iframe via the M key, NVDA reads: This is the iframe title attribute iframe banner landmark navigation landmark open menu button

NVDA identifies the iframe, reads the iframe's title value, drills down through any ARIA role information, and reads the first element in the iframe. It does not read the title of the iframe content.

Tab (browse mode)

Tab goes to the iframe container itself even when no tabindex is present, and no content is read. Pressing Tab again moves focus to the first focusable element and NVDA reads: This is the iframe title attribute iframe banner landmark navigation landmark open menu button

On the second Tab press NVDA reads the iframe's title value, identifies the iframe, drills down through any ARIA role information, and reads the first element in the iframe. It does not read the title of the iframe content.

Tab (forms mode)

Tab goes to the iframe container even if no tabindex is present and NVDA reads: This is the iframe title attribute iframe this is the iframe title attribute document.

NVDA reads the iframe's title attribute, identifies the iframe, speaks the iframe's title attribute a second time, then identifies the content of the iframe as a document.

Recommendation

NVDA identifies iframes and iframe content differently depending on how users navigate to the iframe. Since users will have different experiences based on how they access the content, refraining from using iframes is recommended.
